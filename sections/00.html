Section 0: 처음 보기, 환경 준비, 실행 도구 준비

예상 60-90분 웹사이트 정독 PowerShell + uv 노트북 실행 전 필수

무엇을 배우나요?

Section 0은 AI 개념을 배우기 전에 모두가 같은 출발선에 서도록 만드는 단계입니다. 수업에서 사용할 실행 기준은 Windows PowerShell, uv, Python 3.11, Jupyter Notebook입니다. Python은 브라우저에서 직접 설치하는 것을 기본 경로로 삼지 않습니다. uv가 필요한 Python 실행 환경과 패키지를 준비하고, 우리는 uv run --python 3.11 ... 형식으로 실행합니다.

이 방식을 쓰는 이유는 학생마다 이미 설치된 Python, Anaconda, VS Code kernel, PATH 설정이 다를 수 있기 때문입니다. 전역 python 명령이 되느냐보다, 수업에서 안내한 uv 명령이 되는지가 기준입니다. 따라서 python --version만 실패한다고 바로 Python을 다시 설치하지 않습니다. 먼저 uv --version과 uv run --python 3.11 python --version을 확인합니다.

API key는 내 코드가 OpenAI 모델을 사용할 수 있게 해주는 비밀번호 같은 값입니다. key는 웹사이트나 채팅에는 입력하지 않습니다. 이번 자료에서는 학생이 열어야 할 파일을 줄이기 위해 각 노트북의 API key 입력 셀에 직접 붙여넣습니다. 단, key가 들어간 노트북은 그대로 공유하지 않고, 제출하거나 화면 공유하기 전에는 key 문자열을 지웁니다.

노트북 실습 전에 실행할 것

ZIP을 받은 뒤 압축을 풀고, PowerShell에서 압축을 푼 루트 폴더로 들어갑니다. 루트 폴더에는 START_HERE.md, requirements.txt, notebooks/가 보여야 합니다. 아래 명령으로 uv가 Python 3.11과 수업용 library를 준비한 Jupyter를 엽니다. 

cd amsl-internship-study
uv --version
uv run --python 3.11 python --version
uv run --python 3.11 --with-requirements requirements.txt --with notebook jupyter notebook notebooks

Jupyter가 열리면 00_start_here.ipynb부터 실행합니다. 이 노트북은 루트 requirements.txt를 찾고, 필요한 package import를 확인하고, API key를 붙여넣은 뒤 첫 연결 테스트를 합니다. VS Code에서 직접 열 경우에도 커널이 수업용 uv/Python 3.11 환경인지 확인합니다.

Section 0 결과 확인
  • uv --version이 동작합니다.
  • uv run --python 3.11 python --version이 Python 3.11 실행 환경을 보여줍니다.
  • 00_start_here.ipynb가 루트 requirements.txt 위치와 설치 기준 package를 출력합니다.
  • API key 입력 셀에 수업용 key를 붙여넣은 뒤 smoke test가 짧은 한국어 연결 성공 메시지를 출력합니다.
  • key가 들어간 노트북은 채팅, GitHub, 제출물에 그대로 공유하지 않습니다.
  • 오류를 질문할 때는 실행한 명령, 현재 폴더, 오류 메시지, 모델명만 공유할 수 있습니다.
