Section 2: 답변을 정해진 형식으로 받기

예상 60-90분 Structured output Pydantic LangChain parser

핵심 개념

자연어 답변은 사람이 읽기에는 좋지만, 프로그램의 다음 단계로 넘기기에는 불안정합니다. 필드 이름이 바뀌거나, 값이 빠지거나, 숫자가 문자열로 들어오면 다음 코드가 깨질 수 있습니다. Structured output은 모델 답변을 정해진 구조로 받는 방식이고, 이번 수업에서는 Pydantic으로 그 구조를 검사합니다.

Pydantic은 Python type hint를 사용해 데이터 구조와 validation 규칙을 정의하는 도구입니다. BaseModel은 어떤 필드가 필요한지 정하고, Field는 값의 범위나 설명을 붙입니다. 예를 들어 confidence가 0보다 작거나 1보다 크면 validation error가 나야 합니다. 이 오류는 실패라기보다 “모델 출력 또는 schema 계약을 다시 봐야 한다”는 디버깅 신호입니다.

중요한 구분이 있습니다. Pydantic validation은 형식을 검사하지만 사실성을 검사하지 않습니다. source_id와 quote 모양이 맞더라도, quote가 실제 원문에 없다면 사실성은 틀린 것입니다. Section 2에서는 형식을 안정화하고, Section 3에서 원문 근거를 확인합니다.

LangChain은 여기서 무엇을 하나요?

LangChain은 LLM application의 부품을 연결하는 프레임워크입니다. 이번 Section에서는 LangChain 전체를 깊게 배우지 않고, PromptTemplate, model call, output parser가 각각 어떤 역할을 하는지만 봅니다. PromptTemplate은 입력 문서와 형식 지시문을 합쳐 모델에게 보낼 prompt를 만들고, output parser는 모델이 만든 JSON 문자열을 Pydantic 객체로 바꿉니다.

LangChain을 쓴다고 답이 자동으로 좋아지지는 않습니다. 연결 도구는 prompt와 parser를 다루기 쉽게 해주지만, 여전히 schema 설계, evidence 확인, 답이 없을 때의 정책은 사람이 정해야 합니다.
