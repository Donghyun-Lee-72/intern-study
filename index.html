Agentic AI 실습형 학습 포털

이 수업은 AI 모델을 직접 불러와 보고, 답변을 확인하고, 근거가 있는 작은 AI 프로그램을 만들어보는 섹션형 실습입니다. 처음부터 모든 용어를 알 필요는 없습니다. 웹사이트에서 설명을 읽고, 노트북에서 코드를 실행하고, 각 Section 하단의 결과 확인을 열어 맞게 실행됐는지 비교하면 됩니다.

처음이면 Section 0부터 Python 기본 문법이면 시작 가능 Windows PowerShell + uv 웹사이트는 교재 노트북은 실행용

학습 섹션

이 페이지가 교재입니다. 설명은 웹사이트에서 읽고, 실행은 ipynb 노트북에서 합니다. 노트북은 수업 내용을 실행해보는 도구이고, 웹사이트는 왜 그 코드를 실행하는지 이해하는 학습 문서입니다. 각 Section은 “읽기 → 노트북 실행 → 결과 확인” 순서로 구성되어 있습니다.

처음부터 RAG, Pydantic, LangChain, agent를 외우지 않습니다. 먼저 “AI에게 요청한다 → 답변 형식을 확인한다 → 문서 근거를 붙인다 → 여러 단계를 연결한다”는 흐름을 잡고, 각 도구 이름은 필요한 Section에서 하나씩 연결합니다.
0. 환경 준비 1. AI 호출 2. 형식 검증 3. 문서 근거 4. Workflow 5. Linux 특강 6. 문서 분할 7. Vector 검색 8. Retriever 평가 9. Vector RAG 10. RAG 평가 11. Tool과 Agent 12. 미니 프로젝트
실습 파일 ZIP 다운로드
처음 시작할 때는 실습 ZIP 하나만 받으면 됩니다. ZIP 안에는 루트 requirements.txt와 Section 0-12 노트북과 확장 실습용 데이터가 들어 있습니다. 실행용 Python script 묶음은 두지 않고, 웹사이트에서 학습하고 ipynb에서 실습하는 구조로 단순화했습니다.

API 키와 예상 비용

이 탭은 AI 모델을 쓰기 전에 알아야 할 “키와 사용량” 안내입니다. API는 내 코드가 OpenAI 모델에게 요청을 보내고 답을 받는 방식입니다. 요청할 때 사용하는 비밀번호 같은 값이 API key입니다. 이 key는 웹사이트나 채팅에 입력하지 않고, 각 노트북의 API key 입력 셀에만 붙여넣습니다.

왜 비용이 생기나요?

모델에게 보내는 입력과 모델이 생성하는 출력은 token이라는 사용량 단위로 계산됩니다. 처음에는 token을 “AI가 읽고 쓴 글의 양을 세는 단위” 정도로 이해하면 됩니다. 글을 많이 보내거나 긴 답변을 많이 받으면 사용량이 늘고, 사용량에 따라 비용이 생깁니다.

이번 수업은 제공받은 수업용 API key를 사용하는 흐름을 기준으로 합니다. 학생 개인 결제카드를 등록하지 않습니다. 별도로 안내받지 않았다면 개인 OpenAI 계정에 결제수단을 추가할 필요가 없습니다.
Section 0-4는 실제 생성 모델을 호출합니다. Section 5-6, 8, 10-12의 필수 실습은 API를 호출하지 않고, Section 7과 9는 기본 실습을 오프라인으로 실행한 뒤 선택적으로 작은 실제 API 검증을 수행합니다. 모델 응답 길이와 재실행 횟수에 따라 사용량이 늘 수 있으므로 실패한 셀만 다시 실행합니다.

비용 계산은 어떻게 보나요?

실제 금액은 사용하는 모델의 최신 가격표에 따라 달라집니다. 기본 계산식은 다음처럼 이해하면 됩니다. 

예상 비용 = input token 수 × input token 단가 + output token 수 × output token 단가
  • mini 계열 모델을 쓰면 학생 1명이 Section 노트북을 한 번씩 실행하는 비용은 작게 유지하는 것을 목표로 합니다.
  • 반복 실험, 긴 문서, 여러 번 재실행을 포함하면 비용이 늘어날 수 있습니다.
  • 정확한 금액은 실제 모델의 input/output token 단가와 재실행 횟수에 따라 달라집니다.

계산 예시

아래는 계산 방법을 보여주기 위한 가상의 단가 예시입니다. 가격표가 input 100만 token당 0.20달러, output 100만 token당 0.80달러라고 가정해 보겠습니다. 한 번 실행에서 input 3,000 token, output 1,500 token을 사용했다면 다음처럼 계산합니다. 

input 비용  = 3,000 / 1,000,000 × 0.20달러 = 0.0006달러
output 비용 = 1,500 / 1,000,000 × 0.80달러 = 0.0012달러
총 비용     = 0.0018달러

실제 단가는 수업에서 제공받은 모델의 최신 가격표를 기준으로 확인합니다. 예시 단가를 실제 청구액으로 보면 안 됩니다.

문제 해결

키 오류

OPENAI_API_KEY가 비어 있거나 잘못된 경우입니다. 현재 노트북의 API key 입력 셀에 수업용 key를 붙여넣고 그 셀부터 다시 실행합니다.

모델 오류

gpt-5.4-mini는 현재 검증된 수업용 기본 모델명입니다. 계정에서 사용할 수 없으면 제공된 저비용 모델명으로 OPENAI_MODEL을 바꿉니다.

패키지 오류

루트 폴더에서 uv run --python 3.11 --with-requirements requirements.txt --with notebook jupyter notebook notebooks로 Jupyter를 다시 엽니다. 00_start_here.ipynb의 requirements 확인 셀이 통과해야 합니다.

결과가 흔들림

AI 출력은 약간 변할 수 있습니다. 정해진 형식 확인 실습에서는 validation을 통과하는지가 더 중요합니다.
