# 가상 AMSL 프로젝트 온보딩 안내

이 문서는 문서 분할 실습을 위해 새로 작성한 가상의 안내문이다. 실제 연구실 규정이나 개인정보를 포함하지 않는다.

## 계정과 API key

수업용 API key는 개인별로 전달받아 사용한다. API key는 비밀번호와 같은 비밀정보이므로 공개 GitHub 저장소, 채팅방, 발표 화면에 올리지 않는다. 노트북을 공유하거나 제출하기 전에는 key 문자열과 실행 출력을 다시 확인한다. key가 노출된 것으로 의심되면 문자열만 지우고 끝내지 말고 담당자에게 알려 폐기와 재발급 여부를 확인한다.

## 질문과 오류 공유

오류를 질문할 때는 API key 자체를 보내지 않는다. 대신 실행한 명령, 현재 작업 폴더, 노트북 파일명과 셀 위치, 오류 메시지, 사용한 모델명을 공유한다. Windows PowerShell과 WSL Ubuntu shell은 서로 다른 실행 환경이므로 어느 shell에서 실행했는지도 함께 적는다.

오류 메시지는 마지막 한 줄만 잘라 보내기보다 처음 발생한 예외부터 확인할 수 있는 범위를 포함한다. 다만 사용자 이름이 포함된 절대 경로나 개인 파일명은 필요한 경우 가려서 공유한다.

## 문서와 출처

RAG 실습에서는 답변보다 먼저 검색된 문서를 확인한다. 각 문서는 source_id를 가지며, 문서를 나눈 각 조각은 같은 source_id와 서로 다른 chunk_id를 가져야 한다. 답변에 인용문을 포함할 때는 인용문이 실제 검색된 조각에 존재하는지 확인한다. 형식이 올바른 JSON이라고 해서 내용까지 사실인 것은 아니다.

검색 결과에 답이 없으면 모델이 상식으로 내용을 채우지 않게 한다. 이 경우 not_answered 상태를 사용하고, 질문이 애매하거나 판단이 필요한 경우 needs_review 상태로 보낸다. 검색 실패와 답변 생성 실패를 구분할 수 있도록 단계별 로그를 남긴다.

## 실행 환경

기본 실습 환경은 Windows PowerShell, uv, Python 3.11, Jupyter Notebook이다. 전역 python 명령이 실패한다고 Python을 바로 다시 설치하지 않는다. 먼저 uv --version과 uv run --python 3.11 python --version을 확인한다. 같은 컴퓨터에서도 VS Code가 선택한 kernel이 다르면 설치된 package가 다르게 보일 수 있다.

Linux와 WSL2는 이후 개발 환경을 확장하기 위한 선택 경로다. WSL 안에서 Windows 파일은 보통 /mnt/c 아래에 보이지만, 프로젝트를 어느 파일시스템에 둘지는 도구 성능과 작업 방식을 고려해 정한다. sudo나 apt가 필요한 작업은 목적을 확인한 뒤 수행한다.

## 실습 결과 기록

각 Section을 마치면 실행 성공 여부만 기록하지 않는다. 어떤 입력을 사용했고, 어떤 결과를 예상했으며, 실제 결과가 어떻게 달랐는지 적는다. 모델 출력은 실행할 때마다 조금 달라질 수 있으므로 문장 일치보다 schema, 근거, 답 없음 처리 조건을 확인한다.

문서 분할 실습에서는 chunk 개수, 길이 분포, 잘린 문장, 반복된 문장을 비교한다. 가장 좋은 chunk 크기를 미리 정답으로 가정하지 않는다. 이후 검색 평가 질문을 실행해 어떤 설정이 실제 질문에 더 적합했는지 판단한다.
